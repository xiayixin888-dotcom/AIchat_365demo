# 私域助手 - 朋友圈管理模块 PRD

## 1. 文档概述
*   **产品定位**：为企微客服/选房师提供一个中心化的朋友圈分发与互动管理工作台。
*   **核心目标**：解决多账号朋友圈动态聚合查看的问题，提升点赞、评论等互动行为的处理效率，并结合客户画像实现高效的私聊转化。
*   **目标用户**：一线运营人员、选房顾问、客服人员。

## 2. 整体信息架构与布局
页面采用经典的三栏式（或左右分割结构）：
*   **左侧菜单/筛选区**：提供全局动态的过滤选项（如：全部动态 / 待处理互动 / 城市切换）。
*   **中栏动态流**：以“朋友圈动态内容”为维度聚合展示的卡片列表。
*   **右侧详情区**：选中单条动态后，展示该动态对应的“点赞客户管理”与“评论互动区”。
*   **右侧浮层（抽屉面板）**：全局通用的客户详情组件，包含档案、画像、行为时间线。

## 3. 详细功能与交互说明

### 3.1 动态列表流 (中栏)
**功能目的**：高效浏览各账号发布的朋友圈内容，避免相同内容刷屏。

*   **内容聚合逻辑**：不同客服账号在相近时间内发布的**相同图文内容**，在列表中聚合成一条卡片展示。
*   **发布账号展示规则**：
    *   **视觉规范**：所有企微客服账号昵称后，统一强制追加橘色 `@365淘房` 企业标识。
    *   **折叠逻辑**：卡片顶部仅展示首个发布的账号昵称，若有多个账号，后缀展示为“等N个账号发布”（不换行，不超出卡片宽度）。
    *   **交互状态**：鼠标悬停（Hover）在“等N个账号发布”上时，弹出浮层列表，展示所有参与发布的完整客服账号明细。
*   **卡片内容区**：
    *   展示文案全文（超长时提供展开/收起）、九宫格配图（点击可放大预览）。
    *   底部统计区域展示去重后的真实点赞数与评论数。

### 3.2 点赞管理模块 (右侧上部)
**功能目的**：快速识别对朋友圈感兴趣的客户，并支持一键批量触达。

*   **数据过滤与展示规则**：
    *   **防干扰过滤**：自动过滤并隐藏企微客服账号给自己的点赞，**仅展示真实的外部客户点赞**。
    *   **头像列表**：网格状排列展示客户头像与昵称。若昵称过长，允许在卡片内部换行展示，绝对不可将外层容器横向撑开导致页面左右滚动。
    *   **互动状态标记**：若运营已在评论区或私聊中回复过该客户，其头像右下角叠加展示蓝色的“已回评论”小图标。
*   **批量触达交互**：
    *   **快捷勾选**：提供“全选”与“选择未评用户”的快捷复选框。
    *   **私聊回复按钮**：选中≥1个客户时按钮高亮，点击唤出批量私聊发送弹窗（支持输入文本+插入图片）。
    *   **结果反馈**：发送成功后，系统在工作区顶部 Tab 栏，为选中的每一位客户自动创建或激活私聊会话页面。

### 3.3 评论管理模块 (右侧下部)
**功能目的**：集中处理评论，降低多账号来回切换的认知成本。

*   **发布账号过滤栏 (左栏)**：
    *   **列表展示**：展示参与发布该条聚合动态的所有客服账号。名称完整展示（不使用省略号，不换行），增加列表宽度以保证可读性。
    *   **筛选交互**：
        *   **默认状态**：未选中任何账号，右侧展示该聚合动态下的**全部评论内容**。
        *   **选中状态**：点击某个账号（整行高亮加粗带背景色），右侧评论区实时过滤，**仅展示该账号对应的朋友圈底下的评论与回复**。
        *   **取消选中**：再次点击已选中的账号，取消筛选，恢复展示全部评论。
*   **评论线程流 (右栏)**：
    *   **排序控制**：右上角提供“按时间”和“按热度（回复数量）”切换按钮。
    *   **无效线程折叠（核心防噪）**：若企微客服自己在朋友圈发布了一条评论（如“欢迎私聊”），**且没有任何客户回复**，则该线程自动在前端折叠隐藏；只有当有客户参与回复时，该线程才正常展示。
    *   **回复交互**：
        *   鼠标悬停在某条评论上，右侧出现“回复”按钮。
        *   点击唤出内联输入框。输入框支持插入表情（Emoji面板）和本地图片上传。
        *   提交后，回复内容实时追加至该评论线程末尾。

### 3.4 客户详情抽屉 (右侧全局浮层)
**功能目的**：在不中断当前浏览心流的情况下，快速透视客户意向，辅助沟通话术。

*   **唤起方式**：在点赞区或评论区，左键单击客户头像/昵称，从右侧滑出固定宽度的抽屉面板。
*   **功能配置（朋友圈专属缩减版）**：
    *   *注：与私聊会话不同，朋友圈场景下的客户抽屉剔除了“快捷话术/流转记录/聊天记录”功能。*
    *   **客户档案 (Tab 1)**：白底圆角卡片流。展示业务定义的固化字段（如：意向区域、预算、户型偏好），右上角支持手动刷新数据。
    *   **客户画像 (Tab 2)**：
        *   顶部展示客户头像、昵称、企微在线/授权状态。
        *   **基础信息**：手机号、base_id、unionid、788ID等（需提供一键复制 Icon）。
        *   **标签组**：语料识别标签（总价/面积偏好等）与 CDP 标签池。
        *   **跟进备注**：时间线形式展示运营手工记录的备注信息。
        *   右下角常驻蓝色 AI 助手悬浮按钮。
    *   **客户行为 (Tab 3)**：时间轴样式。记录客户在小程序的页面浏览时长、点击特价房列表等微观行为。

### 3.5 全局快捷动作 (右键菜单)
*   **触发区域**：朋友圈模块内所有的客户头像（点赞区、评论区）。
*   **交互说明**：
    *   右键单击头像，唤起上下文菜单 `[发私聊消息]`。
    *   点击后，系统精确获取该客户的 ID 与基础信息，并在工作区顶部 Tab 栏：
        1. 若该客户的私聊 Tab 已存在，直接切换高亮并跳转至该会话。
        2. 若不存在，则自动新建该客户的私聊 Tab，写入初始客户档案并跳转，实现无缝衔接。

## 4. 异常边界处理
*   **空数据状态 (Empty States)**：
    *   **无点赞**：点赞区展示灰色的心形 Icon 及“暂无点赞”文案。
    *   **无评论**：评论区展示灰色的对话 Icon 及“暂无评论”文案。
    *   **筛选无结果**：当左侧选中某个发布账号，但该账号下暂无有效评论时，展示“当前账号下暂无可展示的评论内容”。
*   **图片加载失败**：
    *   头像或朋友圈配图加载失败时，需提供默认占位图（Fallback Image），避免页面布局塌陷。
*   **长文本截断**：
    *   非关键长文本（如单行标签）超出容器宽度时，使用 `...` 截断，Hover 提示完整内容。
    *   评论正文等需阅读文本，允许自然换行，限制最大高度，超长时显示“展开/收起”按钮。
*   **防抖与节流**：
    *   批量私聊发送、评论提交等写入操作，需增加 Loading 状态与防抖（Debounce）处理，防止重复提交。

## 5. 权限与可见性控制
*   **数据权限**：
    *   **普通选房顾问/运营**：仅能查看分配给自己企微账号的朋友圈动态，以及属于自己客户的互动数据。
    *   **主管/管理员**：可查看团队内所有账号的朋友圈聚合流与互动明细（下拉框支持切换视角或选择“全看”）。
*   **操作权限**：
    *   仅当前动态的**实际发布账号归属人**，才能执行“回复评论”和“发私聊消息”操作。
    *   非归属人查看他人动态时，回复按钮与批量私聊按钮置灰（Disabled），Hover 提示“暂无操作权限”。
*   **企业标识管控**：
    *   `@365淘房` 标识由系统层级控制，基于账号属性（企微客服类型）自动追加，不允许运营人员在前端手动修改或删除。
